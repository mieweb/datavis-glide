# Code Standards

1.  Code is indented with tabs. Each tab is 2 visual spaces.
2.  When changing code, prioritize leaving lines untouched (don’t
    reformat lines, even if they don’t meet these standards).
3.  Write in an object-oriented style.
4.  Don’t [cuddle your else
    keywords](http://wiki.c2.com/?CuddledElseBlocks).
5.  Combine declarations for variables without initializers; use
    separate declarations for variables with initializers.
